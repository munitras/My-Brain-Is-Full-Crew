# Draft production-ready schemas
### Epic 1: The JSONL Message Bus
#### 1. The Strict Message Schema
Field operators must configure the LLM tool outputs to adhere strictly to this JSON schema for every message appended to the bus.
**File reference:** `Meta/schemas/message-schema.json` (For agent prompt context)
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Inter-Agent Message",
  "type": "object",
  "properties": {
    "timestamp": {
      "type": "string",
      "format": "date-time",
      "description": "ISO 8601 timestamp (e.g., 2026-03-25T08:30:00Z)"
    },
    "from": {
      "type": "string",
      "enum": ["architect", "scribe", "seeker", "sorter", "connector", "librarian", "transcriber"]
    },
    "to": {
      "type": "string",
      "enum": ["architect", "scribe", "seeker", "sorter", "connector", "librarian", "transcriber"]
    },
    "status": {
      "type": "string",
      "enum": ["pending", "resolved"]
    },
    "intent": {
      "type": "string",
      "description": "Short, snake_case action descriptor (e.g., file_note, create_area, link_graph)"
    },
    "payload": {
      "type": "object",
      "description": "Key-value pairs of context required for the task. Values must be strings or string arrays."
    },
    "resolution": {
      "type": "string",
      "description": "Required ONLY when status is 'resolved'. Describes the outcome."
    }
  },
  "required": ["timestamp", "from", "to", "status", "intent", "payload"]
}
```
#### 2. The Initial Message Bus File
This is the file the agents will actually read and write to.
**File reference:** `Meta/agent-messages.jsonl`
```jsonl
{"timestamp": "2026-03-25T08:00:00Z", "from": "system", "to": "architect", "status": "resolved", "intent": "initialize_bus", "payload": {"status": "online"}, "resolution": "JSONL bus initialized successfully."}
```
#### 3. The `jq` Wrapper Script (`poll-queue.sh`)
This script isolates the agent from token bloat by pushing the queue filtering down to the CPU.
**File reference:** `scripts/poll-queue.sh`
```bash
#!/bin/bash
# Enforce strict error handling
set -e
# Target file
QUEUE_FILE="Meta/agent-messages.jsonl"
# Check dependencies
if ! command -v jq &> /dev/null; then
    echo '{"error": "jq is not installed on the host system."}'
    exit 1
fi
# Validate input
AGENT_NAME=$1
if [ -z "$AGENT_NAME" ]; then
    echo '{"error": "Agent name parameter is required."}'
    exit 1
fi
# Check if queue exists
if [ ! -f "$QUEUE_FILE" ]; then
    echo '{"error": "Queue file not found at '$QUEUE_FILE'."}'
    exit 1
fi
# Extract and output ONLY pending messages for the requested agent
# The -c flag ensures output remains compact JSONL format
jq -c -M "select(.to == \"$AGENT_NAME\" and .status == \"pending\")" "$QUEUE_FILE" || echo '{"error": "Failed to parse JSONL."}'
```
> **Implementer Note:** Ensure this file is made executable (`chmod +x scripts/poll-queue.sh`).
#### 4. The Human Observability Dashboard
To replace the old Markdown message board, create a new note for the user to monitor the agents. This uses DataviewJS to parse the `.jsonl` file.
**File reference:** `Meta/Dashboard.md`
```javascript
```dataviewjs
// Fetch the raw JSONL file
const rawData = await app.vault.adapter.read("Meta/agent-messages.jsonl");
// Split by line, filter out empty lines, and parse JSON
const messages = rawData.split('\n')
    .filter(line => line.trim().length > 0)
    .map(line => JSON.parse(line));
// Filter for pending messages to keep the dashboard clean
const pendingMessages = messages.filter(m => m.status === "pending");
// Render the table
dv.table(
    ["Time", "From", "To", "Intent", "Payload"],
    pendingMessages.map(m => [
        moment(m.timestamp).format("YYYY-MM-DD HH:mm"),
        m.from,
        m.to,
        m.intent,
        JSON.stringify(m.payload)
    ])
);```
```
